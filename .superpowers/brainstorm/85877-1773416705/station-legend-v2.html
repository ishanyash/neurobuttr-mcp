Station Design System v2

Shape = action type (all equilateral). Color = who did it.

User — Warm palette
Agent — Cool palette (each tool gets its own color)
Design rationale
Warm vs Cool — User actions use warm tones (white, gold, amber). Agent actions use cool tones (green, red, cyan). This creates an instinctive who distinction without dimming either side.
Semantic colors — Green for edits (changes), red for bash (execution risk), cyan for search (exploration), gray-dashed for reads (passive). Colors match developer intuition.
Equal weight — All stations are the same size. Nothing is muted. Every action matters.

● Warm = you    ● Edit    ■ Bash    ▲ Search    ○ Read    ◆ Decision    ◆ Insight
